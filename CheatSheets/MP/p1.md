### Таблица. Методика проверки заполнения табличных списков (MaxPatrol SIEM 26.2)

| № | Наименование проверки | Действие (шаги) | Ожидаемый результат |
|---|---|---|---|
| 1 | **Проверка создания и структуры табличного списка** | 1. В **Консоли управления** перейти в раздел **Настройки** → **Табличные списки**.<br>2. Нажать кнопку **Создать**.<br>3. В поле «Имя» указать: `Test_Block_List`.<br>4. Добавить столбец с именем `ip_address` (тип `String`) и столбец `reason` (тип `String`).<br>5. Нажать **Сохранить**. | Список `Test_Block_List` появился в дереве списков слева. В карточке списка отображаются два столбца: `ip_address` и `reason`. |
| 2 | **Проверка ручного заполнения списка** | 1. В списке `Test_Block_List` нажать кнопку **Добавить запись**.<br>2. В поле `ip_address` ввести `192.168.10.55`.<br>3. В поле `reason` ввести `Test_Manual_Entry`.<br>4. Нажать **Сохранить**. | В таблице появилась новая строка. Значения в ячейках соответствуют введенным данным (`192.168.10.55` и `Test_Manual_Entry`). |
| 3 | **Проверка импорта данных из CSV-файла** | 1. Создать файл `import.csv` со следующим содержимым:<br>`ip_address,reason`<br>`10.0.0.5,Imported_Row_1`<br>`10.0.0.6,Imported_Row_2`<br>2. В интерфейсе списка `Test_Block_List` нажать кнопку **Импорт**.<br>3. Выбрать файл `import.csv`, настроить соответствие столбцов.<br>4. Запустить импорт. | Записи из файла успешно добавлены. В интерфейсе отображаются строки с IP `10.0.0.5` и `10.0.0.6`. Ошибок кодировки нет. |
| 4 | **Проверка автоматического заполнения из правила корреляции** | 1. Перейти в **Настройки** → **Правила корреляции**.<br>2. Создать правило с именем `Rule_Auto_Fill`.<br>3. Условие: `msg contains "AUTO_ADD_TEST"`.<br>4. Действие: **Добавить в табличный список**.<br> - Список: `Test_Block_List`.<br> - Столбец `ip_address` = `src_ip` (из события).<br> - Столбец `reason` = `"Auto_Added"`. <br>5. Сгенерировать событие с текстом `AUTO_ADD_TEST` и `src_ip = 172.16.5.20`. | В списке `Test_Block_List` автоматически появилась запись: `ip_address` = `172.16.5.20`, `reason` = `Auto_Added`. |
| 5 | **Проверка использования списка в условии правила** | 1. Создать правило корреляции `Rule_Check_List`.<br>2. Условие: `src_ip IN Test_Block_List.ip_address`.<br>3. Действие: Завершить работу (или создать инцидент).<br>4. Сгенерировать событие с `src_ip = 10.0.0.5` (есть в списке).<br>5. Сгенерировать событие с `src_ip = 1.1.1.1` (нет в списке). | Правило сработало только для события с `src_ip = 10.0.0.5`. Для события с `src_ip = 1.1.1.1` правило не сработало. |
